# Product

## Register

product

## Users

Mindustry mod authors and tool-minded developers who are editing JSON content locally. They are usually working with a running backend, class metadata, field paths, and exported mod files, often while debugging whether a class or field is valid.

## Product Purpose

MindustryMIT helps users create, inspect, edit, sync, and export Mindustry mod JSON without hand-writing every field path. Success means a user can connect to the local backend, initialize or fetch docs, create a class instance, add fields with confidence, and export usable JSON with fewer mistakes.

## Brand Personality

Calm, precise, forgiving. The interface should feel like a dependable developer tool: direct language, clear state, compact controls, and no decorative noise.

## Anti-references

Do not make it look like a marketing landing page, a game-themed skin, a raw protocol console, or a dense IDE clone. Avoid decorative gradients, oversized cards, vague buzzwords, modal-heavy flows, and protocol labels as primary user-facing copy when a clearer action label exists.

## Design Principles

- Keep the main workflow visible: connect, load docs, create an instance, add fields, inspect output, export.
- Explain state at the point of action, especially disconnected, empty, pending, synced, and error states.
- Prefer standard product controls that users already understand.
- Keep backend protocol details available but secondary to the editing task.
- Protect local drafts and make destructive actions visually distinct.

## Accessibility & Inclusion

Target WCAG 2.2 AA for contrast, focus visibility, keyboard access, reduced motion, and readable Chinese and English labels. The tool should work for long editing sessions without relying on color alone to communicate state.
