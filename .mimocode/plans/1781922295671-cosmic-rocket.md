# Plan: AllClass 支持指定父类过滤

## Goal
在 `AllClass` WebSocket 接口中支持可选的 `Parent_Class` 参数，返回指定父类的所有子类（包含父类自身）。

## 改动文件

### 1. `src/main/java/com/mindustry/ide/tool/json/JsonParser.kt`

- 添加 `isSubclassOf(className, parentClass)` 私有方法：递归遍历父类链判断继承关系
  - 运行时路径：用 `Class.isAssignableFrom()`
  - 文档路径：遍历 `classDocs` 的 `parentType` 链
- 添加 `getAllClassesByParent(parentClass: String)` 方法：从 `getAllClasses()` 结果中过滤出指定父类的子类（含父类自身）
- 在 `IJsonParser` 接口中添加 `getAllClassesByParent` 声明

### 2. `src/main/java/com/mindustry/ide/tool/json/JsonApi.kt`

- 修改 `AllClass` handler（约 line 440）：从 `data.content` 中手动解析可选的 `Parent_Class` 字段
  - 如果 `Parent_Class` 为空或不存在 → 调用 `getAllClasses()`（向后兼容）
  - 如果 `Parent_Class` 非空 → 调用 `getAllClassesByParent(parentClass)`

## 设计说明
- `Parent_Class` 不加入 `WebSocketDataType.AllClass` 的 `input` 定义，避免破坏现有客户端
- 在 handler 中手动解析 content JSON 中的可选字段，向后兼容
- 过滤结果包含父类自身
- 父类匹配复用 `classKeys()` 同时支持简单类名和全限定名
